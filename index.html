HEURISTICS GUIDE

USABILITY

Heuristics

A curated guide to Jakob Nielsen's Usability Heuristics. These ten broad rules of thumb define the standard for human-computer interaction. Hover over each cell to explore the logic behind seamless user experiences.

01.
UH#01
VISIBILITY OF SYSTEM STATUS

The design should always keep users informed about what is going on through appropriate feedback within a reasonable time.

02.
UH#02
MATCH BETWEEN SYSTEMS AND REAL WORLD

The design should speak the users' language. Use words, phrases, and concepts familiar to the user, rather than internal jargon.

03.
UH#03
USER CONTROL AND FREEDOM

Users often choose system functions by mistake and will need a clearly marked "emergency exit" to leave the unwanted state.

04.
UH#04
CONSISTENCY AND STANDARDS

Users should not have to wonder whether different words, situations, or actions mean the same thing.

05.
UH#05
ERROR PREVENTION

Even better than good error messages is a careful design which prevents a problem from occurring in the first place.

06.
UH#06
RECOGNITION RATHER THAN RECALL

Make objects, actions, and options visible. The user should not have to remember information from one part of the dialogue to another.

07.
UH#07
FLEXIBILITY AND EFFICIENCY OF USE

Accelerators — unseen by the novice user — may often speed up the interaction for the expert user.

08.
UH#08
AESTHETIC AND MINIMALISTIC DESIGN

Dialogues should not contain information which is irrelevant or rarely needed. Every extra unit of information competes with the relevant units.

09.
UH#09
HELP USERS RECOGNIZE AND RECOVER

Error messages should be expressed in plain language (no codes), precisely indicate the problem, and constructively suggest a solution.

10.
UH#10
HELP AND DOCUMENTATION

Even though it is better if the system can be used without documentation, it may be necessary to provide help and documentation.
